# Repository Guidelines

## Project Overview

`@dhzh/foundry` is an AI-native local developer runtime for tools, skills, agents, and workflows. It is a TypeScript package published as both ESM and CJS, with declarations generated by `tsdown`.

## Layout

- `src/index.ts` is the public package entrypoint.
- `test/` contains Vitest tests.
- `dist/` is generated build output and should not be edited by hand.
- Root config files define TypeScript, ESLint, Vitest, Commitlint, and tsdown behavior.

## Tooling

- Use `pnpm` for all package operations.
- The expected package manager is `pnpm ^11.7.0`.
- The expected runtime is `node ^26.3.0`.
- Do not add JavaScript dependencies with `npm` or `yarn`.

## Common Commands

- `pnpm run dev` runs the source entrypoint in watch mode.
- `pnpm run build` builds `dist/` with declarations.
- `pnpm run lint` runs ESLint.
- `pnpm run lint-fix` applies ESLint fixes.
- `pnpm run test` runs the Vitest suite once.
- `pnpm run test:dev` runs Vitest in watch mode.
- `pnpm run test:coverage` runs tests with coverage.

## Code Style

- Write TypeScript using ESM syntax.
- Keep strict TypeScript settings satisfied; avoid weakening types to silence errors.
- Prefer small, explicit exported APIs from `src/index.ts`.
- Preserve the package's side-effect-free behavior unless a feature explicitly requires otherwise.
- Keep generated artifacts out of manual edits.

## Tests

- Add or update tests in `test/` for behavior changes.
- Use Vitest's `it` and `expect` style consistently with existing tests.
- Prefer focused tests that exercise public behavior through package exports.

## Commits

- Use English commit messages.
- Follow Conventional Commit style; commitlint extends `@commitlint/config-conventional`.

## Before Finishing

- For code changes, run at least `pnpm run lint` and `pnpm run test` when practical.
- Run `pnpm run build` when changing exports, TypeScript configuration, or build settings.
