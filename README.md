# @dhzh/foundry

An AI-native local developer runtime for tools, skills, agents, and workflows.

Foundry is published as a CLI package. The CLI starts a local Hono server and
serves the bundled React Web UI from `dist/web/`.

## Requirements

- Node.js `^24.18.0`
- pnpm `^11.9.0`

## Setup

```sh
pnpm install
```

## Development

Start the Rsbuild development server for the Web UI:

```sh
pnpm run dev:web
```

The browser opens at [http://localhost:3000](http://localhost:3000).

Build the Web UI into `dist/web/`, then run the CLI in watch mode:

```sh
pnpm run build:web
pnpm run dev:cli
```

Other useful commands:

```sh
pnpm run build       # Build dist/cli/ and dist/web/
pnpm run build:cli   # Build only the CLI bundle
pnpm run lint        # Run ESLint across the workspace
pnpm run test        # Run Vitest once
```

## Run the CLI

Build the package and launch the published executable wrapper:

```sh
pnpm run build
node bin/index.js
```

The CLI serves the built Web UI at
[http://127.0.0.1:7777](http://127.0.0.1:7777).

## Web UI

The private [`web`](packages/web/package.json) workspace uses:

- [Rsbuild](https://rsbuild.rs) 2
- React 19
- React Compiler through `@rsbuild/plugin-react`
- TypeScript with bundler module resolution

Rsbuild is configured in
[`packages/web/rsbuild.config.ts`](packages/web/rsbuild.config.ts). Its
production output is written directly to the repository-level `dist/web/`
directory consumed by the CLI.

## Project Layout

| Path | Role |
|------|------|
| `src/cli/` | CLI entrypoint and Hono static server |
| `packages/web/` | Rsbuild + React Web UI workspace |
| `packages/web/src/index.tsx` | Browser entrypoint |
| `packages/web/src/app.tsx` | Root React component |
| `packages/web/rsbuild.config.ts` | React Compiler and Web build configuration |
| `bin/index.js` | Published `foundry` executable wrapper |
| `dist/cli/` | Bundled CLI output generated by tsdown |
| `dist/web/` | Web UI production output served by the CLI |
| `specs/plans/` | Numbered implementation plans and historical design context |
