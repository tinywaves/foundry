# @dhzh/foundry

An AI-native local developer runtime for tools, skills, agents, and workflows.

Foundry is distributed as a CLI package. It starts a local Hono server and
serves the bundled React Web UI from `dist/web/`.

The Web UI is built with Astryx and currently provides a minimal React 19
starter surface using the Astryx neutral theme.

## Requirements

- Node.js `^24.18.0`
- pnpm `^11.9.0`

## Setup

```sh
pnpm install
```

## Development

Start the Web UI development server with Rsbuild:

```sh
pnpm run dev:web
```

Rsbuild opens the browser at [http://localhost:3000](http://localhost:3000).

To run the CLI against the local Web UI, build the static assets first and then
start the CLI in watch mode:

```sh
pnpm run build:web
pnpm run dev:cli
```

The CLI serves the Web UI at [http://127.0.0.1:7777](http://127.0.0.1:7777).

## Commands

Run these commands from the repository root:

```sh
pnpm run build          # Build the CLI and Web UI
pnpm run build:cli      # Build only dist/cli/
pnpm run build:web      # Build only dist/web/
pnpm run dev:cli        # Run the CLI from source in watch mode
pnpm run dev:web        # Start the Rsbuild Web UI development server
pnpm run lint           # Run ESLint
pnpm run test           # Run Vitest once
pnpm run test:coverage  # Run Vitest with coverage
```

## Run A Production Build

Build the package and launch the executable wrapper:

```sh
pnpm run build
node bin/index.js
```

To use the published package without cloning the repository:

```sh
pnpm dlx @dhzh/foundry
```

The `foundry` executable listens on `127.0.0.1:7777` and prints a clickable
local URL when it starts.

## Web UI

The private [`web`](packages/web/package.json) workspace uses:

- [Rsbuild](https://rsbuild.rs) 2
- React 19
- React Compiler through [`@rsbuild/plugin-react`](https://rsbuild.rs/plugins/list/plugin-react)
- Astryx components and the neutral theme
- TypeScript with bundler module resolution

The Web entrypoint is
[`packages/web/src/index.tsx`](packages/web/src/index.tsx), which imports the
global Astryx reset and theme styles from
[`packages/web/src/index.css`](packages/web/src/index.css). The root component
lives in [`packages/web/src/app.tsx`](packages/web/src/app.tsx).

Rsbuild is configured in
[`packages/web/rsbuild.config.ts`](packages/web/rsbuild.config.ts). Production
assets are written directly to the repository-level `dist/web/` directory so
the CLI can serve them next to its bundled output.

## Project Layout

| Path | Role |
|------|------|
| `specs/plans/` | Numbered implementation plans and their naming convention |
| `src/cli/` | CLI entrypoint and Hono static server |
| `packages/web/` | Rsbuild + React Web UI workspace |
| `packages/web/src/index.tsx` | Browser entrypoint |
| `packages/web/src/app.tsx` | Root React component |
| `packages/web/src/index.css` | Astryx reset and theme imports |
| `packages/web/rsbuild.config.ts` | React Compiler and Web build configuration |
| `bin/index.js` | Published `foundry` executable wrapper |
| `dist/cli/` | Bundled CLI output generated by tsdown |
| `dist/web/` | Web UI production output served by the CLI |
